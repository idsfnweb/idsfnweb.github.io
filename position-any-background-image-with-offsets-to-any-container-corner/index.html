What's the problem?

It'll be an easy task if I told you to position a background image, 0.4em from the left border & 1em from the top border. Right ?
The word "OR NAH." is a background image.

but, what about 1em from the bottom border and 0.2em from the right border ?

Of course, you'll have an answer, I did too. Sadly it wasn't the best.

I used to calculate the width and the height of the box, then I use subtraction.
In our example 1em from the bottom means box height - 1em.

That's totally okay, but in responsive design this solution is useless.

My next solution was to insert this background image as an actual <img />, so I could play with the layout. It works all the time, but adding extra markup just to solve a CSS issue is a big no-no.

Often times, the design is much complex than our example and because of that, I always end up with a very mangled and ugly HTML code.
Also if I wanted in the future to change the style, I'll need to go back and read the markup and ask myself why I did certain things in a certain way.

In CSS3, we're able to solve this problem so easily. First, we'll learn how then we'll practice by creating this album cover.

1. flexible background positioning

To control the position of any background image, we use the property background-position.

These 3 syntax below are quite popular, but they are not capable of accepting an offset.

background-position: bottom left;  
background-position: 10% 20%;  
background-position: 1em 2em;

The first value is for the x-axis and the second is for the y-axis.

The initial value of background-position is 0%.

background-position: 0% 0%;

1.1 background-position can accept offsets

We can express 1em from the bottom border and 0.2em from the left border as

background-position: bottom 1em left 0.2em;

for some reasons this syntax is not mentioned at MDN. I don't know why!

now we can easily switch it back to any corner with the exact measurement we want, all we need to do is change the keyword left to right

background-position: bottom 1em right 0.2em;

1.2 calc function

If i said background-position: bottom right;, That means 100% bottom and 100% right and I could express it like this

background-position: 100% 100%;

What if I could subtract the value that I'm aiming for as an offset.
the function calc can do this for me.

background-position: calc(100% - 1em) calc(100% - 0.2em);

as a result, we'll have something like this

1.3 Background origin & Padding

Remember, last episode when we used background-clip, the concept is similar.

for example

background-position: top;

We know that each CSS element has for boxes

so what box do top refer to?

border box?

Nop, by default it refers to the padding box, using background-origin we can change it to content-box.

background-position: bottom right;  
background-origin: content-box;

now we can add padding, to achieve what we want.

box-sizing: border-box;  
padding: 0 0.2em 1em 0

as a result

As you see I've used border-box to make sure any extra padding will not increase or change the box size.

1.4 Who's better?

The first method is more readable, but let's say we don't know the offset and we can only calculate it via javascript at render time. 'calc' will suit better.

There's no who's best unless we're talking about performance and to be honest, I don't know who performs better.

2. Let's practice

Last month I released Nuclin

Nuclin is a soundtrack album, there's no need for me to be concerned about including the Explicit flag into my album cover.

But you know what, let's redesign the album cover and add the Explicit flag to it.

first download the cover from here, and let's start.

<div class="card">  
  <img src="" alt="Nuclin Album cover" />
  <div class="topLayer"></div>
</div>

The <img /> is our cover.
.topLayer will hold the Explicit flag.

Everything is wrapped inside .card,so adding more elements will be easier in the future.

in our CSS, I'm gonna start with this

.card {
  width: 40%;
}

img {  
  max-width: 100%;
}

let's quickly add a little bit of box-shadow to the img
Tweak the shadows to your taste, this is what I came up with

Back to .topLayer, as I said we'll put the explicit flag as a background-image in this div.

First, we need .topLayer to be in the same .card size.

.topLayer {
  background-color: blue;
  position: absolute;
  top: 0;
  left: 0;
  bottom: 0;
  right: 0;
}

I'm trying to stretch .topLayer to it nearest relative, then I'm painting the whole layer with blue.
Now before I check if the whole .card is covered with blue, I must specify who's the relative

.card {
  width: 40%;
  position: relative;
}

Here we go

You can remove the blue background and instead insert the Explicit flag.

background: url(http://bit.ly/2gc3vRn) no-repeat;  
background-position: calc(100% - 0.4em) calc(100% - 1em);

to make the bottom of our album cover blend well with the white background. I will add an extra layer on top of everything

<div class="shine"></div>  

.shine {
  position: absolute;
  top: 0;
  left: 0;
  bottom: 0;
  right: 0;
}

By default, the relative is body and that's exactly what we want.

I'm going to make this layer white at the bottom and transparent at the top

.shine {
  background: linear-gradient(180deg, transparent, white);
}

That's our final result

You can make the blend less obvious by lowering the opacity of .shine
