2Questions to constantly ask while writing CSS! EP1

12 November 2016 on CSS

Writing DRY and maintainable, flexible code is a huge topic and squeezing it only in 1 post is quite hard.
For that reason, I'll teach you two questions you should ask yourself whenever you write any CSS code, so your code will be in a much better state.

Wait, Why should I care?

If you are working on a big project, you'll suffer. Easy things will start to get hard to tweak because all you have is a mangled, hard to manage code.
Writing code that looks GOOD and performs GOOD is what you should aim for all the time.
Let's focus today on "Don't Repeat Yourself" AKA DRY.

Q1: Is it scalable?

What if I change font-size: 18px; to font-size: 45px;
do you think other related effects like drop-shadow will scale ? Absouletely not!

Shortly we will introduce redundant code.

to fix it, switch from px to % or em or rem.

Yeah, I do that all the time, that's responsive design basics!

We usually sleep on effects, and we think it won't make that much difference.

Q2: Does it do only one particular thing?

Each function must do only one particular job without mutating the state.

Same idea of Functional programming, to keep it simple ignore mutate and replace function with class.

in reality, it's impossible to create 100% functional CSS for the whole project.
Just like code coverage, nobody can get it to a 100%.

This where we left our code previously

Now, with some changes

I managed the old code into new small classes.
Each one of them performs a specific job. All classes are more likely now to get re-used in any new button in our project. That way no code duplication will be introduced in the future.
For now, I only have buttons, but you can go deeper than the example, and be much precise.

Less code via responsive design

The more responsive your design is, the few media queries you'll use.

Here are some points you should pay attention to

  • Always use %, ems, vw or vh

  • Aim for max-width when you want a fixed width for large resolution, so it'll adapt automatically to smaller resolutions.

  • Set any replaced elements such as img, object, video, iframe to max-width: 100%;.

  • Scale your images via object-fit or background-size: cover;.

  • Use FlexBox for your layout.

If you find yourself with so many media query rules, go back and re-examine your code.
